設計規範 Guildline

Ciaoyu Web UI Kits 是由 Canboo (Tsai. Cheng Peng) 根據以往製作各種類型網站的經驗，將所有網頁常見的元素進行系統性的整理，並透過與數位視覺設計的討論，建立一個依附於 Bootstrap 的網站設計規範，便於專案協同伙伴能有一套建置標準進行開發。

框架版本 Framework Version

  • jQuery v3.3.1 (https://jquery.com/)
  • Popper v1.14.6 (https://popper.js.org/)
  • Bootstrap v4.2.1 (https://getbootstrap.com/)
  • Font Awesome Free v5.6.3 (https://fontawesome.com)

設計原則 Design Principle

易用性

UI 必須以大眾習慣的方式來設計，元件彼此之間的距離必須能夠讓使用者清楚辨別關連性，且外觀必須能夠清楚知道物件是否可點擊。

多螢體驗

元件在規劃時都必須符合 Responsive Web Design 的需要，且考慮使用該裝置的情境，讓使用者在不同載具使用時都能得到一定水準的體驗。

解析度

元件與圖形盡量使用 HTML/CSS 或 SVG 打造，特殊字型使用 Webfont 技術，重點圖片如 brand 應提供多尺寸備用，讓任何解析度下的網站都能夠以完美呈現。

可動性

規範能讓協作伙伴針對專案取用共同的色彩、元件、以及程式碼，但針對需求允許修正並非鐵則，如果發現是規範不足之處，貢獻新的設計可加速未來專案開發時程。

HTML架構 HTML Markup

  • 請將 <meta charset="utf-8" /> 置放在 <head> 的第一行，避免英文之外的語系在部份舊的瀏覽器中出現亂碼。
  • 若非必要，建議 <script> 請置放於 <body> 區塊的最後。

瀏覽器 Browser support

瀏覽器支援度主要依據瀏覽器核心對於 CSS 的支持程度，若要支持過舊的版本往往需要選擇折衷方案導致不能使用更快更有效率的技術，比如說響應式設計(Responsive Web Design)、陰影(shadow)、以及圓角(border-radius)等。
